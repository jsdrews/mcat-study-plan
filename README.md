# MCAT Study Plan

Research-backed study planner for the MCAT. Generates a personalized week-by-week schedule based on evidence about what actually works: spaced repetition, retrieval practice, optimal study hours, and practice test timing.

**[View the plan →](https://jsdrews.github.io/mcat-study-plan/study_plan.html)**

## What it does

Set your test date and start date, and the planner builds a phased schedule:

| Phase | % of prep | What happens |
|-------|-----------|--------------|
| Content Review | ~40% | Topic rotation across all 4 MCAT sections + Anki + daily CARS |
| Practice & Integration | ~40% | Timed sections, mixed question sets, full-length practice tests |
| Test Readiness | ~20% | Final full-lengths, targeted weak areas, taper before test day |

Every day is planned. Every week has a guaranteed rest day. No day exceeds 5 focused hours (except full-length test days).

## Key research findings

- **4–6 focused hours/day** is optimal — quality drops sharply beyond 6h ([Residency Advisor](https://residencyadvisor.com/resources/mcat-prep/the-10-hours-a-day-mcat-study-myth-optimal-time-by-outcome))
- **Retrieval practice** (flashcards, practice questions) yields 61% retention at 1 week vs 40% for rereading ([Roediger & Karpicke, 2006](http://psychnet.wustl.edu/memory/wp-content/uploads/2018/04/Roediger-Karpicke-2006_PPS.pdf))
- **Spaced repetition** users were 2x more likely to pass medical entrance exams ([PMC](https://pmc.ncbi.nlm.nih.gov/articles/PMC12255137/))
- **First 4–5 full-length tests** provide the bulk of score improvement (~+8 points) ([Residency Advisor](https://residencyadvisor.com/resources/mcat-prep/how-many-mcat-practice-exams-correlate-with-maximum-score-gain))

## Quickstart

```bash
# clone and install
git clone git@github.com:jsdrews/mcat-study-plan.git
cd mcat-study-plan
uv sync
```

Edit the config cell in `notebooks/study_plan.ipynb`:

```python
TEST_DATE = date(2026, 9, 12)   # your MCAT date
START_DATE = date.today()        # when you start studying
HOURS_PER_DAY = 5.0              # recommended: 4-6
REST_DAYS = 1                    # rest days per week
```

Then either open the notebook interactively or export to HTML:

```bash
task notebook   # open in Jupyter
task export     # build HTML in docs/
```
